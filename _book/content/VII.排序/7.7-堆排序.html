7.7 堆排序

7.7 堆排序

堆排序在最坏情况下和平均情况下的时间复杂度均与归并排序相同，都为 O(nlogn)O(n \log n)O(nlogn)，但堆排序的空间复杂度为 O(1)O(1)O(1)，虽然堆排序要比空间复杂度为 O(n)O(n)O(n) 的归并算法要稍慢一些，但比空间复杂度为 O(1)O(1)O(1) 的归并算法要快。

首先将待排序的表转换为最大堆，并将堆中的第一个记录和最后一个记录进行交换。由于第一个记录总是最大关键字的记录，所以经过交换后，该记录排在有序的位置上，然后对堆进行重新调整。将上述操作执行 n−1n - 1n−1 遍，即可得出排序正确的表。下面函数 adjust 的参数 list 的 root 结点可能不满足堆的性质，函数将 root 结点与其子结点进行比对并进行调整，使得整个二叉树满足堆的性质。函数 heapsort 实现了堆排序的策略。调用方式为 heapsort(list, n)。

void adjust(element list[], int root, int n)
{
    int child, rootkey;
    element temp;
    temp = list[root];
    rootkey = list[root].key;
    child = 2 * root; // 左孩子
    while (child <= n) {
        if ((child < n) && (list[child].key < list[child+1].key))
            child++;
        if (rootkey > list[child])
            break;
        else {
            list[child / 2] = list[child];
            child *= 2;
        }
    }
    list[child / 2] = temp;
}

void heapsort(element list[], int n)
{
    int i, j;
    element temp;

    for (i = n/2; i > 0; i--)
        adjust(list, i, n);
    for (i = n-1; i > 0; i--) {
        SWAP(list[1], list[i+1], temp);
        adjust(list, 1, i);
    }
}
